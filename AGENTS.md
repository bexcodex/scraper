# AGENTS.md

## Package identity
- npm: `@bexcodex/scraper` — TikTok media metadata/download URL scraper
- ESM-first (`"type": "module"`); tsup emits both ESM + CJS
- Node `>=18.18` required; **npm only** (`packageManager: npm@10.0.0` — do not use pnpm/yarn/bun)

## Commands

| Command | What it runs |
|---|---|
| `npm run build` | `clean → generate:exports → tsup` |
| `npm run typecheck` | `tsc --noEmit` (no emit, type-check only) |
| `npm run generate:exports` | Regenerates `src/index.ts` via `tsx scripts/generate-exports.ts` |
| `npm run clean` | Deletes `dist/` |

**No test runner is configured.** `coverage/` is gitignored but no framework is installed.

## Generated file — never edit by hand
`src/index.ts` is a barrel generated by `scripts/generate-exports.ts`. Editing it directly will be overwritten on the next build.

Run `npm run generate:exports` after any add/rename/delete of `.ts` files under `src/` or `src/scrapers/`.

## Adding a scraper
1. Create `src/scrapers/<name>.ts`, export your function(s) and types.
2. Run `npm run generate:exports`.
3. Done — tsup picks up via `src/index.ts`.

## TypeScript quirks
- `verbatimModuleSyntax: true` — use `import type { X }` for type-only imports, never bare `import { X }`.
- `isolatedModules: true` — no const enums; re-exporting types requires `export type`.
- `moduleResolution: Bundler` — relative imports without extensions are fine.
- `tsup.config.ts` and `scripts/generate-exports.ts` are included in `tsconfig.json` and are typechecked.

## Build output
`dist/` contains four files: `index.js` (ESM), `index.cjs` (CJS), `index.d.ts`, `index.d.cts`. Never commit `dist/`.

## CI
Two workflows in `.github/workflows/`:
- `ci.yml`: `npm ci → typecheck → build` on push/PR to main/master
- `publish.yml`: `npm ci → build → npm publish --access public` on release or `v*.*.*` tag (requires `NPM_TOKEN` secret)
